# YAT — Yet Another Typeless

[繁體中文](./README.md) | [English](./README.en.md)

**想說什麼，就直接說吧。**
YAT 是一個跨平台的桌面語音輸入工具。按住快捷鍵，對著麥克風說話，放開按鍵後，你說的話就會直接化為文字，精準地出現在你的輸入框裡。

不再受限於打字速度，讓你的輸出跟上你的思緒。

## 為什麼選擇 YAT？

作為一個文字工作者、開發者，或是常常需要回覆大量訊息的人，你可能覺得打字太慢，或者腦中的想法總是一閃而過。YAT 專為這些「想得比打字快」的時刻而生。

- ⚡️ **全域可用，即說即打**：不管你在用 Word、Notion、VS Code 還是 Line，按住快捷鍵就能用語音輸入，放開就自動貼上。
- 🧠 **不只是語音辨識，更是你的 AI 編輯**：串接強大的語言模型（如 OpenAI、Groq 等），自動為你**去除贅詞、加上標點符號、整理段落**，甚至排版成條列式。說話結巴？沒關係，YAT 會幫你轉成完美通順的句子。
- 🌍 **雙語切換無障礙**：常常中英夾雜？沒問題！YAT 能完美保留你原本的語感，不會硬生生把你說的英文翻成中文，也不會把中文拼成奇怪的英文。
- 🎯 **專有名詞不再出錯**：提供「詞彙表」功能，你可以自由建立品牌名、人名或專業術語，確保音譯永遠正確。
- 🔒 **隱私優先，BYOK (自備金鑰)**：你的語音與文字只會傳送到你信任並設定的 API 服務商，絕不會經過 YAT 的伺服器。所有的歷史紀錄也都安全地留在你的電腦裡。

## 誰最適合 YAT？

- **開發者**：口述 Commit Message、架構思維、或是快速撰寫 README。
- **內容創作者**：靈感來時立刻口說記錄，不怕靈感游走。
- **重度鍵盤使用者**：想減少手指負擔，讓手腕休息一下。
- **任何人**：只要你覺得能「說完就貼上」會讓生活更輕鬆。

## 快速開始 (只需 3 分鐘)

開始享受語音輸入帶來的便利，只需三個簡單的步驟：

### 1. 取得 YAT
前往 [Releases 頁面](../../releases) 下載適合你系統的版本（主打 Windows、macOS；Linux 維持 Experimental）。安裝後開啟後，你會在選單列或系統匣看到 YAT 的圖示。

> ℹ️ **平台提醒**：YAT 目前以 Windows 與 macOS 為主要支援平台。macOS 若要使用自動貼上或錄音時自動暫停媒體，通常需要先在「輔助使用」允許 YAT；Linux 目前維持 Experimental，錄音時的背景音訊控制需要 `pactl`，自動暫停媒體需要 `playerctl`，Wayland 也可能限制部分能力。

### 2. 連接你的語音服務
YAT 支援所有相容 OpenAI 格式的 API。你可以先只設定語音辨識；文字潤飾是之後想加強時再開啟的選配。
1. 點擊選單列或系統匣圖示進入**設定**。
2. 前往 **語音辨識** 分頁，填入你的 API Base URL、模型名稱與 API Key。
3. 點擊「測試連線」確認設定正確。
*(進階建議：前往 **文字潤飾** 分頁設定另一個端點，讓輸出的文字更乾淨、通順！)*

### 3. 開始說話！
一切準備就緒。將游標停在你想要輸入文字的地方：
- **Windows / Linux**：按住右邊的 `Ctrl` 鍵
- **macOS**：按住右邊的 `Cmd` 鍵

看到螢幕上出現懸浮的錄音提示後，開始說話。說完後放開按鍵，文字就會自動貼上！

> 💡 **小撇步**：怕貼錯地方嗎？你可以在設定中將「輸出模式」改為**僅複製**，YAT 處理完後會直接存入剪貼簿，讓你確認後再手動貼上。

## 深入探索

YAT 不僅僅是個能錄音的按鈕，它還提供許多貼心功能讓你的流程更順暢：

- **完全自訂的快捷鍵**：不習慣按住說話？你可以改成「點擊切換錄音/停止」、「連點兩下」或「組合鍵」（例如 `Ctrl + Shift + C`）。
- **額外參考資訊（進階功能）**：當你真的需要時，可以讓 YAT 在開始錄音時一併參考剪貼簿、選取文字、目前使用的 App、輸入框內容，甚至**畫面截圖**，幫助 AI 更貼近你眼前的工作內容。（為了保護隱私，**這些功能預設全部關閉**，只有你手動開啟的項目才會送出。）
- **更安靜的錄音流程**：錄音時可以選擇維持背景音訊原樣、暫時壓低音量，或直接靜音；也能自動暫停播放中的音樂／影片（依系統支援程度而定）。
- **可攜式備份與移轉**：你現在可以把設定或詞彙表匯出成 JSON，方便備份、換機，或把在 macOS 上調好的配置搬到 Windows（反之亦然）。出於安全考量，API Key 不會包含在匯出檔中。

## 常見問題 (FAQ)

**Q：一定要設定「文字潤飾」嗎？**
A：不用！如果你只設定了「語音辨識」，YAT 會直接輸出語音轉文字的最原始結果。設定「文字潤飾」是為了讓排版更漂亮、文字更精鍊。

**Q：自動貼上失敗怎麼辦？**
A：在某些權限較嚴格的作業系統（例如 macOS 未開啟輔助使用、或 Windows 以系統管理員執行的 app 中），自動貼上可能會失敗。這時建議在設定中把輸出模式改為「**僅複製**」。

**Q：我的 API Key 和歷史紀錄安全嗎？**
A：絕對安全。API Key 直接儲存在你的設備上，轉錄結果也只會存入本地的 SQLite 資料庫（預設保留 30 天）。YAT 沒有任何遙測或收集數據的行為。

### 對開發者：從原始碼建置

如果你想自己動手打造或貢獻程式碼：

**所需環境**
- Node.js 20.19+（或 22.12+）
- Rust 1.77+（`rustup` 安裝 stable toolchain）

**各平台前置系統依賴**
- **Windows**：Microsoft Edge **WebView2 Runtime**（Win11 內建；Win10 可能需手動安裝）、**Microsoft C++ Build Tools**（含 MSVC 與 Windows SDK）。
- **macOS**：**Xcode Command Line Tools**（`xcode-select --install`）。打包 Apple Silicon 目標時另需 `rustup target add aarch64-apple-darwin`。
- **Linux**（Experimental）：`libwebkit2gtk-4.1-dev` `libappindicator3-dev` `librsvg2-dev` `patchelf` `libasound2-dev` `libxdo-dev`；執行期另需 `playerctl`（自動暫停媒體）與 `pactl`（錄音時壓音量）。

```bash
npm install
npm run validate     # tsc + vite build + cargo check + cargo test
npm run tauri dev
npm run tauri build
```
