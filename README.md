# YAT — Yet Another Typeless

[繁體中文](./README.md) | [English](./README.en.md)

> 按住快捷鍵，直接開口。
> 放開之後，文字就到游標位置。

**👉 [下載最新版](../../releases) ｜ [三分鐘上手](#三分鐘上手) ｜ [功能一覽](#功能一覽) ｜ [常見問題](#常見問題)**

YAT 是一個桌面語音輸入工具。你只要按住快捷鍵說話，YAT 就會先把語音轉成文字；如果你有設定「文字潤飾」，它還會順手把內容整理得更乾淨、更像可以直接送出的版本。

## 先看重點

| 如果你想要 | YAT 會怎麼幫你 |
|---|---|
| 更快輸入 | 按住熱鍵說話，放開就完成 |
| 任何 App 都能用 | 直接貼到目前游標位置，或只寫入剪貼簿 |
| 文字更乾淨 | 去掉贅詞、補標點、整理段落和條列 |
| 中英混說也自然 | 保留原本語感，不硬翻成單一語言 |
| 專有名詞更穩 | 用詞彙表固定修正品牌名、產品名和術語 |

## 這些人會用得很順

- 想得比打字快的人
- 想用口述寫訊息、文件、筆記或 commit message 的人
- 常常中英混說的人
- 想減少鍵盤負擔的人
- 想要「說完就貼上」的人

## 三分鐘上手

> **👉 第一次用，先做這三件事：下載、設定語音辨識、開始說話。**

### 1. 先下載安裝

| 平台 | 你要下載什麼 | 說明 |
|---|---|---|
| Windows | `YAT_x.x.x_x64-setup.exe` | 雙擊安裝，可選路徑與安裝範圍 |
| Windows（企業部署） | `YAT_x.x.x_x64_en-US.msi` | 適合 GPO / SCCM 靜默部署 |
| macOS（Apple Silicon） | `YAT_x.x.x_aarch64.dmg` | 開啟後拖到 Applications |
| macOS（Intel） | `YAT_x.x.x_x64.dmg` | 開啟後拖到 Applications |
| Linux（通用） | `YAT_x.x.x_amd64.AppImage` | 下載後 `chmod +x` 直接執行 |
| Linux（Debian/Ubuntu） | `YAT_x.x.x_amd64.deb` | `sudo apt install ./YAT*.deb` |
| Linux（Fedora/RHEL） | `YAT_x.x.x_x86_64.rpm` | `sudo dnf install ./YAT*.rpm` |

- 到 [Releases](../../releases) 下載對應版本
- 安裝後開啟 YAT
- 從系統匣圖示進入設定

### 2. 先把「語音辨識」設好

- 前往「**語音辨識**」分頁
- 選 **Groq** / **OpenAI**，或填入自訂 OpenAI 相容 API 網址
- 貼上 API Key
- 按「**測試連線**」

### 3. 直接開始說話

- 預設熱鍵是 **按住說話**
  - Windows / Linux：**右 Ctrl**
  - macOS：**右 Cmd**
- 按住開始錄音，放開結束
- 如果你還沒填好「**文字潤飾**」的 API Key，YAT 會直接輸出轉錄結果
- 如果你想讓結果更乾淨，再去設定「**文字潤飾**」

> **👉 想先用最穩的方式開始？**
> 把輸出模式設成「**僅複製**」，先確認流程順了，再改回「**自動貼上**」。

## 建議你這樣開始

| 你現在最在意什麼 | 建議做法 |
|---|---|
| 想最快開始 | 先只設定「語音辨識」 |
| 想結果更像可以直接送出 | 再設定「文字潤飾」 |
| 怕貼錯地方 | 輸出模式先用「僅複製」 |
| 常講品牌名或專有術語 | 先把「詞彙表」加好 |
| 想一開機就能用 | 打開「開機自動啟動」 |

## 功能一覽

### 輸入與操作

- **全域熱鍵**：支援按住說話、單一按鍵、連按兩下、組合鍵
- **浮動狀態窗（Capsule）**：顯示錄音狀態、時間與麥克風音量
- **Esc 取消**：錄音中或處理中都能取消
- **系統匣常駐**：平常待命，需要時再開設定

### 輸出方式

| 模式 | 適合什麼情況 |
|---|---|
| **自動貼上** | 想說完就直接出現在游標位置 |
| **僅複製** | 想自己確認後再貼上，或常遇到權限 / 相容性問題 |

- 就算自動貼上失敗，結果還是會留在剪貼簿
- 你可以決定是否一直保留最新結果在剪貼簿

### 文字整理

- **文字潤飾**：去掉贅詞、重複與口吃，補標點、分段、整理條列
- **中英混說保留**：不硬把內容改成單一語言
- **詞彙表**：固定修正品牌名、產品名和專有名詞
- **提示詞 / 自訂規則**：調整輸出語氣與格式

### 日常使用

- **歷史紀錄**：可搜尋、複製、重試、刪除
- **音效提示**：開始、結束、完成、錯誤都有聲音回饋
- **錄音時自動靜音**：Windows / macOS 支援
- **主題切換**：深色 / 淺色 / 跟隨系統
- **雙語介面**：預設為繁體中文，也可切換 English
- **開機自動啟動**：登入後直接待命

## 服務商怎麼選

YAT 使用你自己的 API Key（BYOK）。請求會直接送到你設定的服務商。

| 你想要 | 建議服務商 | 說明 |
|---|---|---|
| 最快開始 | **Groq** | 速度快、入門門檻低、免費額度友善 |
| 直接用 OpenAI | **OpenAI** | 適合想用官方 OpenAI API 的人 |
| 有自己的端點 | **自訂 OpenAI 相容服務** | 可以接你自己的 OpenAI 相容 API |

| 功能 | 常用模型 |
|---|---|
| 語音辨識 | `whisper-large-v3-turbo`、`whisper-1` |
| 文字潤飾 | `llama-3.3-70b-versatile`、`gpt-4o-mini` |

> **👉 想先上手：**
> 先把 **Groq + 語音辨識** 跑順，再決定要不要加上文字潤飾。

## 設定裡可以調什麼

| 分頁 | 你會在這裡做什麼 |
|---|---|
| **總覽** | 看目前是否已經設定完成、最近輸出是什麼 |
| **一般設定** | 調整熱鍵、輸出模式、麥克風、主題、介面語言、音效、自動靜音、等待時間、重試次數、開機自啟 |
| **語音辨識** | 設定服務商、API 網址、API Key、模型和語言提示 |
| **文字潤飾** | 開關 AI 潤飾，並設定服務商、API 網址、API Key 和模型 |
| **提示詞** | 加上簡單規則，或進一步調整系統提示詞 |
| **詞彙表** | 管理品牌名、產品名和常用術語修正 |
| **歷史紀錄** | 搜尋、複製、重試和清理舊紀錄 |

## 各平台先知道

| 平台 | 你要先知道的事 |
|---|---|
| **macOS** | 需要麥克風權限；自動貼上和全域熱鍵通常還需要「輔助使用」權限 |
| **Windows** | 如果目標 App 是系統管理員權限，自動貼上可能失敗；這時用「僅複製」最穩 |
| **Linux** | 全域熱鍵目前需要 X11 或相容層；如果貼上不穩，建議改用「僅複製」；Linux 目前也不支援錄音時自動靜音 |

- 補充：在 Linux 上，按 **Esc** 取消時，該按鍵仍可能同時送到前景 App。

### 安裝與解除安裝

| 平台 | 安裝方式 | 解除安裝方式 |
|---|---|---|
| **Windows（EXE）** | 雙擊 `-setup.exe`，可選「僅我」或「所有使用者」，可選安裝路徑 | 「設定 → 應用程式」中移除，會詢問是否清除使用者資料 |
| **Windows（MSI）** | 雙擊 `.msi` 或透過 `msiexec` 靜默安裝 | 「設定 → 應用程式」中移除 |
| **macOS** | 開啟 `.dmg`，把 YAT 拖到 Applications 資料夾 | 把 YAT.app 拖到垃圾桶（使用者資料留在 `~/Library/Application Support/com.garyellow.yat/`） |
| **Linux（AppImage）** | `chmod +x` 後直接執行，不需要安裝 | 刪除 `.AppImage` 檔案即可 |
| **Linux（deb）** | `sudo apt install ./YAT*.deb` | `sudo apt remove yat`（加 `purge` 可移除系統設定） |
| **Linux（rpm）** | `sudo dnf install ./YAT*.rpm` | `sudo dnf remove yat` |

## 常見問題

| 問題 | 答案 |
|---|---|
| 一定要先設定什麼才可以用？ | 先把「語音辨識」設好就可以開始使用。 |
| 文字潤飾一定要開嗎？ | 不一定。就算你還沒填好 API Key，YAT 也會先直接輸出轉錄結果。 |
| 預設介面語言是什麼？ | 預設是繁體中文，可以在「一般設定」切換成 English。 |
| 預設熱鍵是什麼？ | Windows / Linux 是右 Ctrl；macOS 是右 Cmd，預設為按住說話。 |
| 預設輸出模式是什麼？ | 預設是「自動貼上」，同時也會把結果保留在剪貼簿。 |
| 自動貼上失敗怎麼辦？ | 先改用「僅複製」，通常最穩。 |
| 歷史紀錄會留多久？ | 預設 30 天，可以在設定裡調整。 |
| API Key 會送到 YAT 的伺服器嗎？ | 不會。API Key 會存在你的裝置上，並只送到你設定的端點。 |

## 隱私與資料

- **API Key 只存本機**：會儲存在你的裝置上，並只送到你設定的端點
- **音訊不落地**：錄音只在處理期間存在記憶體中，不會被長期儲存
- **歷史紀錄在本機**：轉錄結果存放在本機 SQLite 資料庫，預設保留 30 天
- **沒有遙測**：YAT 不會回傳使用統計
